## Root Cause / Investigation

- The current list rendered readiness and status as 2.5px colour-only dots — the label lived only in `aria-label` (sr-only) and a hover/focus tooltip, so sighted users had to decode colour or hover each dot. (The round-1 D/P/A and ✓/!/✕/+ glyphs the brief mentioned were not present in the current code; the dots were pure colour.)
- **Decision (labels-next-to-dots AND a legend):** per admin-studio.md "Lists And Tables" (scan-first) and "restrained colour communicates state", a visible short label beside each dot gives per-row at-a-glance decoding without cross-referencing; a compact header legend explains the full vocabulary once. Both, because per-row labels are abbreviated to one word and the legend names every possible state (including ones not on the current page).
- Reused the existing `readiness.label` ("Strong"/"Needs work"/"Blocked"/"Opportunities") and a new marketer-facing `pageStatusLabel` ("Published"/"Draft"/"Archived") — no raw schema values (admin-studio.md "Language").
