## Ordering decision (snapshot must not fail the save)

The draft is persisted by `persistPageEditorDraft` FIRST and commits. The
manual_save snapshot + prune run AFTER, inside
`snapshotManualSaveRevisionSafely`, which catches and logs any error. So if the
snapshot INSERT or the prune RPC fails, the user's save is already durable and
the action still returns `{status:"saved"}` (unit-proven by the
"still reports the save as successful when the snapshot fails" test).
