## Current step structure BEFORE (documented per brief)

The old `NewPageChoiceGate` was a 3-step wizard with a progress indicator ("Create page progress", "Step N of 3") and Back/Continue/Start-building footer actions:

- **Step 1 "Page type":** pick one of 4 page types (resource / blog / landing / video). The only real choice.
- **Step 2 "Starting point":** "Blank page" vs "Template" — but `SeoPageEditorForm` passes no `savedTemplates` (defaults to `[]`), so "Template" was **always disabled** and the step always resolved to Blank. No effective choice.
- **Step 3 "Ready to build":** a read-only summary, then "Start building page" → `onChoosePageTemplate(pageType, templateKey)`.

So the admin clicked Continue → Continue → Start building (3 screens) to commit one real decision. Inputs the wizard collected: page type, and starting-point/template (effectively always blank). Everything else is set in the editor afterward.

## Design (one step)

A single panel: the 4 page-type cards (the one real input) + a single "Start building page" button calling the unchanged `onChoosePageTemplate(selectedPageType, templateKey)`. The starting-point/template input is **not lost**: when `savedTemplates` has entries for the selected type, an optional "Start from a template" group renders inline (Blank + each template), defaulting to Blank — matching the old effective default. With no saved templates (the current wiring), it's a clean one-decision screen. `onChoosePageTemplate` (controller) is untouched — only the path to it changed.
