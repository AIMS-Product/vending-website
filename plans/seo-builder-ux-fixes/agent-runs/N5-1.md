## Root Cause / Investigation

1. `"use server" file can only export async functions, found object` —
   first browser run hit the route error boundary. Root cause: `actions.ts`
   had `export const REDIRECT_STATUS_CODES`. Next 16 forbids non-async value
   exports from a `"use server"` module. Fix: made the constant module-private
   (kept `export type` only — types are erased).
2. Value preservation failed on first inline-error test (inputs blanked).
   Root cause: uncontrolled inputs in a `useActionState` form reset on the
   post-action re-render. Fix: server actions now echo submitted `values`;
   the create/edit forms render them as `defaultValue` and re-key on each
   result so the echoed values re-mount. Isolated re-test confirmed
   `source value: "/about-not-a-builder-path"`, `destination value:
"/blog/keep-this-value"` preserved with `aria-invalid=true` and the inline
   message "Builder redirect sources must be builder page paths."

Design note (disable): the `redirects` table has NO enabled/active column.
Adding one would require a migration touching customer-visible resolution — a
violation of the CORE INVARIANT on a RISKY surface. Per the "edit and/or
enable/disable" acceptance wording, I implemented delete + edit (which fully
covers removal and correction); "disable" maps to delete here. No schema
change made.
