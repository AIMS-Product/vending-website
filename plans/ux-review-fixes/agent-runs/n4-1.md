# N4 — Quick Tour card must not cover the panel it highlights (P1)

## Status

DONE — fix landed, repo gate + browser gate green.

## Root cause

`WalkthroughCard` computed `top = targetRect.bottom + 16`, and when that didn't fit it fell back to `targetRect.top - 180` clamped to `viewportPadding` (16px). For step 1 the target is the full-height left "blocks" rail (`fixed top-32 bottom-4` / `xl:h-[calc(100dvh-7rem)]`), so both branches land the card on top of the rail and block interaction.

## RED / GREEN / REFACTOR

- RED — Wrote `walkthrough-card-position.test.ts` asserting (via a rect-intersection helper) that a full-viewport-height target produces a card placement that does NOT intersect the target, for both left-pinned and right-pinned tall rails, plus two normal-target cases. Confirmed RED by temporarily forcing the tall branch through the old below/above logic: the two tall-target tests failed with `intersects` true (overlap). Reverted.
- GREEN — Added `placeBesideTarget`: when a target is >= 60% of viewport height, place the card to the right of the target (`right + 16px gap`), falling back to the left side if it won't fit, clamped to viewport. Normal targets keep the exact prior below/above + clamp logic. 4/4 tests pass.
- REFACTOR — Split into `placeBelowOrAboveTarget` / `placeBesideTarget` helpers, named constants (`CARD_WIDTH`, `CARD_HEIGHT`, `VIEWPORT_PADDING`, `CARD_GAP`, `TALL_TARGET_RATIO`), exported `WALKTHROUGH_CARD_DIMENSIONS` for the test.

## Residual risk

Low.
