## Design decision

The redirect/panel decision is made **client-side** in the component, not via `redirect()` in the server action. Rationale: `submitPublicLeadAction` is shared by both apply and contact; adding `redirect()` there would (a) throw inside the action (framework behavior) and (b) need branching to avoid redirecting contact. Keeping it client-side leaves the `{status/message/fieldErrors}` contract in `lead-action-handler.ts` completely untouched and keeps the apply-vs-contact branch where the `intent` prop already lives.
