## Root cause

`useState(() => formatStat(parsed, 0))` seeded the displayed value to a zeroed variant. Because the value lives in the component body (not an effect), the server-rendered HTML emitted `<span>0+</span>` / `<span>$0M+</span>`. Real values only existed as serialized client-component props; crawlers, no-JS users, link previews, and first paint saw zeros.

## Fix

Seed state with the real `value` so SSR and first client render both emit the truth (no hydration mismatch ‚Äî server and client first render are identical). The count-up is now pure progressive enhancement: the effect resets to 0 only after hydration, then animates up when the IntersectionObserver fires. If JS is off or the observer never fires, the real value is already in the DOM.
