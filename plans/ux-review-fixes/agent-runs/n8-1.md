## Root cause of the current 404

`/blog/[slug]` is **not** a news route — it is a SEO Page Builder route. Both
`generateMetadata` and the page delegate to
`@/lib/page-builder/public-page-route`, which resolves the slug via
`getPublishedSeoPageByPath("/blog/{slug}")` and calls `notFound()` when no
**builder** page is published at that path.

A valid _news_ slug (e.g.
`how-to-choose-the-perfect-location-for-vending-machine`) has a published row in
`news_posts` but **no** corresponding `seo_pages` row at `/blog/{slug}`, so
`getPublishedSeoPageByPath` returns `null` → `notFound()` → 404. There was no
blog→news mapping of any kind. That is why a valid slug hard-404s today.

## Fix

In `BlogBuilderPage`, before rendering:

1. Resolve the builder page once via `getPublishedSeoPageByPath("/blog/{slug}")`
   (reused read-only from services).
2. If a builder page **exists**, render it unchanged (builder wins — preserves
   current behavior, including the edge case where a slug exists as both a
   builder page and a news post).
3. If **no** builder page exists **and** `hasPublishedPostSlug(slug)` is true
   (reused read-only from `@/lib/services/news`), call
   `permanentRedirect("/news/{slug}")` (Next 16 → 308 Permanent).
4. Otherwise fall through to `renderBuilderPage`, which `notFound()`s → 404 for
   junk slugs.

No hardcoded slug lists. `permanentRedirect` chosen per current Next docs
(`node_modules/next/dist/docs/.../permanentRedirect.md`): serves a 308 in a
Server Component render context, uses the TS `never` type (no `return` needed).
`hasPublishedPostSlug` is short-circuited so it is not called when a builder
page already matches.

## Behavior preservation

- `/news/{slug}` rendering is untouched (no edits under `src/app/news/**`).
- Unknown legacy slugs still 404 (builder `notFound()` fall-through; unit-tested).
- No redirect loop: `/blog/{slug}` → `/news/{slug}`; the news route never
  redirects back to `/blog`, and the redirect only fires when **no** builder
  page exists, so it cannot bounce.
- Builder pages published at `/blog/{slug}` still render (builder-first;
  unit-tested).
