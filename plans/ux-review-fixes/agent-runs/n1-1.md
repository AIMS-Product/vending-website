### Note on test file extension

The team-lead brief gave `AdminShell.test.tsx` as an example. The repo's `vitest.config.ts` uses `include: ["src/**/*.test.ts"]`, which does NOT match `.tsx`. A `.test.tsx` file is not collected even when passed explicitly on the CLI (verified: "No test files found"). Every existing test in the repo — including the only other admin component test, `SeoPageCommentsPanel.test.ts` — uses `.test.ts` with `renderToStaticMarkup`. I matched that actual convention rather than widen the shared config (out of scope). Test behavior is identical.
