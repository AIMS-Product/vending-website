# N12 — Public accessibility sweep

**Status:** COMPLETE — all three items addressed, repo + browser gates green.

## Per-item disposition

### Item 1 — Video testimonial focus indicators + "possible focus trap"

**Focus indicator: REPRODUCED → FIXED.**
RED (tab-walk, before): every `<video>` testimonial tile on `/` and case-study
tile on `/case-studies` received keyboard focus with no visible indicator —
`visibleFocus=false`, the only outline being the browser default `auto/1px`
(invisible against the busy poster + brutalist card). Stops affected on `/`:
Thomas Rohlader, Joe Natoli, Jason Priest, Anthony CEO (4 tiles); same 4 on
`/case-studies`.
GREEN: added an explicit on-brand ring to the `<video>` elements
(`focus-visible:ring-4 focus-visible:ring-[#066a99] focus-visible:ring-offset-2`)
plus a global `:focus-visible` outline floor in `globals.css`. After tab-walk:
each video's primary stop now reports `visibleFocus=true` with the
`#066a99` ring + white offset. Screenshot:
`shots/n12-home-video-focus-after.png` (vs `-before.png`),
`shots/n12-casestudies-video-focus-after.png`.

Note: each `<video controls>` still produces 2–3 native tab stops (the element
plus internal media-control sub-elements in its shadow DOM). The sub-control
stops cannot be styled by page CSS and were never the flagged element — the
review flagged the video element itself, which is now fixed.

**Focus trap: NOT REPRODUCED.**
Scripted tab-walk confirmed focus advances cleanly through all four video tiles
and exits the group: from the last video (Anthony) the next Tab lands on the
"Apply Now" link, then continues into the footer nav. No element captures or
loops focus. The review's "possible focus trap" flag was a false positive
caused by the browser's native multi-stop media controls.

### Item 2 — No skip-to-content link anywhere: ADDED

RED: tab-walk found no skip link as any tab stop; the contract test
(`src/app/accessibility-landmarks.test.ts`) failed before the layout edit.
GREEN: added `<a href="#main-content" class="skip-to-content">Skip to content</a>`
as the first child of `<body>` in `src/app/layout.tsx`, gave `<main>` the id
`main-content`, and styled `.skip-to-content` in `globals.css` (visually hidden
via `translateY(-150%)`, revealed on `:focus`). It is now the first tab stop on
every public page; activating it sets `#main-content` and moves focus past the
header (next Tab lands inside `#main-content`, not the header nav). The reveal
renders as an on-brand brutalist orange button with the blue focus ring.
Screenshots: `shots/n12-home-skiplink-after.png`,
`shots/n12-home-skiplink-zoom.png`, `shots/n12-casestudies-skiplink-after.png`.
