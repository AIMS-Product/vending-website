## ATTEMPT 2 (redispatch — orchestrator violation report)

VIOLATION FIXED:
Attempt 1 called the sibling setter INSIDE the other setter's updater
function (setIsDesktopSeoSidebarCollapsed inside
setIsDesktopBlockSidebarCollapsed's updater and vice versa). React updaters
run during the render phase and must be pure; a setState inside one is a
side effect (StrictMode double-invokes updaters) — the same bug class the I1
walkthrough fix (commit 56851ac) removed from BuilderEditorWalkthrough.

REWIRE (useSeoPageEditorController.ts, toggle functions only):
The transition is now computed in the event handler from the closure state
values (isDesktopBlockSidebarCollapsed / isDesktopSeoSidebarCollapsed — both
fresh every render since the toggle functions are recreated per render),
then issued as two plain setter calls side by side, which React batches:

const next = nextDesktopPanelCollapseState({
thisCollapsed: isDesktopBlockSidebarCollapsed,
otherCollapsed: isDesktopSeoSidebarCollapsed,
});
setIsDesktopBlockSidebarCollapsed(next.thisCollapsed);
if (next.otherCollapsed !== isDesktopSeoSidebarCollapsed) {
setIsDesktopSeoSidebarCollapsed(next.otherCollapsed);
}

Mirrored for toggleSeoSidebar. No updater functions remain in either desktop
branch. The narrow-editor (mobile) branches are byte-identical to main.
nextDesktopPanelCollapseState and all attempt-1 tests kept exactly as they
were (accepted by orchestrator). The P1-9/I11 comments were moved onto the
handlers, with an explicit note that the transition must never be a setState
inside another setter's updater.
