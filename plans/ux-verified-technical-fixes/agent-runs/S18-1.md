## Root cause (confirmed, matches S16)

On `/admin/pages/new`, two **named `region` landmarks** nest:

- AdminShell's content wrapper `<section aria-labelledby="admin-shell-title">`
  (`src/components/admin/AdminShell.tsx`)
- the create-gate panel `<section aria-labelledby="new-page-choice-title">`
  (`SeoPageEditorShell.tsx:76-78`)

A `<section>` with an accessible name is exposed as a `region` landmark, so two
same-role landmarks with no distinguishing unique label exist on the page →
axe `landmark-unique` (moderate) fires on `.min-w-0` (the outer AdminShell
section).

## Fix (GREEN — smallest markup change)

`SeoPageEditorShell.tsx`, `NewPageChoiceGate`:

- Changed the inner panel `<section aria-labelledby="new-page-choice-title">` →
  a plain `<div>` (dropped the `aria-labelledby`, so it is no longer a `region`
  landmark). Matching closing `</section>` → `</div>`.
- Kept every class string identical, kept the visible `<h2 id="new-page-choice-title">`
  heading element (structure/anchor unchanged), kept all copy, layout, and the
  `onChoosePageTemplate` behavior. The recently-rewritten N13 one-step gate is
  not regressed.

After the fix only one region landmark remains on the route (AdminShell's), so
the rule is satisfied.
