# S6 — Case-studies heading order (findings C053/C086)

## Problem

`/case-studies` renders the page `h1` in `CaseStudiesHero`, then `CaseStudyVideos`
and `CaseStudyQuotes` whose only headings were per-card `h3`s. axe `heading-order`
fires on the jump from `h1` straight to `h3` with no intervening `h2`.
Outline before: `h1 → h3(×4 videos) → h3(×10 quotes) → h2(FinalCta)`.

## Fix (smallest viable, visuals identical)

Added a screen-reader-only `<h2 class="sr-only">` at the top of each section so the
card `h3`s sit under a proper `h2`. No visible heading exists in the design for
these grids, so sr-only matches how the site already hides structural text
(e.g. the `sr-only` star rating in `CaseStudyQuotes`/`Testimonials`).

Section labels added as typed content (`caseStudySectionHeadings`) per the project
rule that page copy traces to `src/lib/content/`:

- Videos: "Video case studies"
- Quotes: "Written testimonials"

Outline after: `h1 → h2(sr) → h3(×4) → h2(sr) → h3(×10) → h2(FinalCta)` — sequential.

Did not touch `CaseStudyVideos`/`CaseStudyQuotes` card structure, the recent
`h-full` equal-height cards, or avatars.
