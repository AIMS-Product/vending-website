# S7 — News cards visible on first paint (C005)

## Status

DONE (acceptance criteria already met by current code; added a regression guard so it stays met).

## Summary

The C005 finding describes news cards rendering as "blank outlines until you
scroll" and attributes it to a scroll-reveal animation. Investigation shows
**no scroll-reveal animation has ever existed** on `/news`. Current code already
renders all cards fully visible on first paint with no scroll. The original
"blank outline" symptom in the persona evidence was actually a **failed/unloaded
cover image** (broken-image alt text), which is exactly what the already-landed
C030 `ImageWithFallback` fix resolves. No source change to `NewsList` was made or
warranted; introducing `opacity-0` + IntersectionObserver would create the very
blank-on-first-paint defect this node exists to prevent (explicitly listed as an
unsafe outcome).

## Scope / files changed

- `src/components/sections/NewsList.test.tsx` — added one test
  ("renders cards visible on first paint (no scroll-reveal hidden state)")
  asserting the article markup contains no `opacity-0` / `invisible` /
  `data-reveal="false"`. Locks in the C005 invariant.
- No change to `src/components/sections/NewsList.tsx` (no reveal logic exists to fix).
- Evidence screenshots under `plans/ux-verified-technical-fixes/agent-runs/shots/`.

### GREEN

No production fix needed — behavior already correct. Added the regression test

### REFACTOR

None.

## Root cause (of the original C005 report)

Misattributed cause. Original evidence screenshot
`reports/ux-persona-review/screenshots/news-001-load.png` shows the third card
displaying raw broken-image alt text ("Top 10 Profitable Products to Stock...")
with no image — a **failed cover image**, not a hidden card. The cards themselves
were always rendered. Personas read the missing images as "blank outlines" and
guessed "scroll-reveal animation." The real failure mode (broken/unloaded cover
images) is handled by the C030 `ImageWithFallback` change (branded placeholder),
which is already on this branch. Current `/news` confirms: cards are visible with
content on first paint at every viewport, even with images stalled.

## Behavior preservation + evidence

- ImageWithFallback (C030) untouched; its test suite still green (2 passed above).
- NewsList rendering (data, pagination, card content) untouched.
- No reveal animation removed (none existed); acceptance criterion "below-fold
  cards still reveal/animate on scroll" is N/A because there is no reveal
  animation in this codebase — cards render directly. This does not regress any
  prior intended behavior.

## Residual risk

Very low. The only change is a test addition. If the team genuinely wants a
deliberate below-fold reveal animation as a new feature, that is net-new scope
(not a C005 fix) and would need its own node with mount-time in-view handling to
avoid reintroducing the blank-first-paint defect.

## Recommendation

Mark S7 / C005 as resolved. The acceptance criteria (first-row cards visible on
first paint, no scroll required) are satisfied by current code; the regression
guard prevents a future scroll-reveal from breaking it. The original "blank
cards" symptom is covered by the C030 ImageWithFallback fix already on-branch.
