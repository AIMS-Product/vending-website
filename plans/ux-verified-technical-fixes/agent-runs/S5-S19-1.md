  `type="submit"` with `form="news-editor-form"` and `name="intent"` /
  `value="save"|"publish"`, so they submit the EXISTING editor form via the

    correct escape for the stacked-aside layout and matches the proven pattern.

  `pb-24 lg:pb-0` so the fixed bar never overlaps the last fields on mobile, and
  mounted `<NewsMobileSaveBar formId="news-editor-form" />` after `</form>`
