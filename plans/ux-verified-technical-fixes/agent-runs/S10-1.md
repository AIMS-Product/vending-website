## Goal recap

`/admin/news` rows previously exposed only an Edit affordance; retiring a post
required opening its editor and using the in-editor Archive intent. Ported the
proven SEO-pages-list pattern: a per-row actions menu with Archive behind a
confirmation, plus row checkboxes with a bulk archive bar. Archive is the
existing reversible status change (`status -> "archived"`); no hard delete.

- `returnTo` is allowlisted to `/admin/news` / `/admin/news?...`, so a forged

  both actions; bulk caps at 50 ids and reports overflow as failed.

  archived list is paginated (7/page) — a freshly archived post may land on a
  later archived page, which is expected list behavior, not a defect.
