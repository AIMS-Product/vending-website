## Behavior verified

When `/admin/pages` renders, every genuinely interactive element in the list
rows exposes a hit area ≥24×24px (WCAG 2.5.8), and the compact visual density
is unchanged.

## Which elements are interactive (and which are NOT)

The status/readiness dots are the headline concern in the brief, but they are
**decorative**, not interactive. `StatusBadge` and `LegendGroup` render each dot
as `<span class="size-2.5 rounded-full" aria-hidden="true" />` inside a
non-interactive `<span>` — there is no button, anchor, or role on or around
them. The Playwright audit confirms this: `interactiveDots: 0` out of 10 dots.
Per the brief and WCAG, decorative elements get **no** hit area, so they were
left untouched (and their 10px visual size is preserved).

Genuinely interactive row elements measured:

| Status/readiness dots (`size-2.5`, 10px) | No — `aria-hidden` spans | n/a             | n/a                              | Not touched (correctly)                          |

all 5 visible rows; every row produced identical checkbox numbers (16→24).

## The fix

Only the bulk-select checkbox failed. It must keep its 16px visual box (compact
aesthetic is a stated user preference), so the fix enlarges the _hit area_ only:
wrap the `<input>` in a `<label>` that adds `p-1` (4px each side → 16+8 = 24px
square) and cancel the padding's layout effect with negative margins so density
and the original `mt-1` alignment are preserved.

```tsx
<label className="-mx-1 mt-0 -mb-1 flex shrink-0 cursor-pointer items-start p-1">
  <span className="sr-only">Select {page.title} for bulk actions</span>
  <input
    type="checkbox"
    name="ids"
    value={page.id}
    form="bulk-archive-form"
    className="size-4 rounded border-slate-300 text-[#0b63f6] focus-visible:ring-2 focus-visible:ring-[#0b63f6]/35 focus-visible:outline-none"
  />
</label>
```

Geometry: label `mt-0` + `p-1`-top (4px) lands the checkbox top at +4px — the
same as the old `mt-1`. `-mx-1` + `p-1`-left lands its left edge at 0px (same as
before). `-mb-1` keeps the row from growing. The `name`/`value`/`form` attrs the
bulk-archive form relies on are unchanged; the accessible name moved from an
`aria-label` on the input to an `sr-only` label span (the `<label>` associates
it). A native checkbox can't take padding reliably, so the label-wrapper is the
correct technique rather than padding the `<input>` directly.
