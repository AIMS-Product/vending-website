### Measurement protocol

True cold loads. For each page: rebuild once (`npm run build`), then for EVERY
page start a fresh `npx next start -p 3000`, measure the first-ever hit on that
server, kill the server. The Playwright harness
(`tools/measure-cls.mjs`, driven by `tools/run-cls.sh`) uses a fresh browser
context per page with the HTTP cache disabled (CDP `Network.setCacheDisabled`),
a buffered `layout-shift` PerformanceObserver that excludes `hadRecentInput`
entries, and an 8s settle after `networkidle`. Default viewport 1280×720.

## Root cause (from attribution, not assumed)

The before-fix attribution showed a SINGLE shifting element on every failing
page: the `<footer>`, moving `prev {y:510,h:210}` → `curr {y:0,h:0}` at
~250–600ms. The value `0.29167` (= 7/24) was bit-identical across pages and
**unchanged by the font fix** — i.e. a deterministic structural reflow, not a
font swap.

Probing the cold SSR HTML for `/about` proved the mechanism: the response
contains `aria-busy="true"` (the root `loading.tsx` Suspense fallback) with the
`<footer>` at byte ~10,651, while the full page is ~42,800 bytes. So the
streamed shell paints the short spinner fallback first, the footer lands high on
the page (~y510), then the real (taller) content streams in and shoves the
footer down — the recorded shift.

The prior round's hypothesis ("footer streams in late") was wrong: the footer IS
in the cold HTML. The footer is the victim; the cause is the root Suspense
fallback being shorter than the real content.

## Changes (smallest fix per measured cause)

1. **`src/app/loading.tsx`** — added `min-h-[100svh]` to the fallback container.
   The root `loading.tsx` is the streaming mechanism (Next auto-wraps the root
   layout's `children` in `<Suspense fallback={<Loading/>}>`). Reserving a full
   viewport of height keeps the footer below the fold from the first paint, so
   swapping in real content no longer moves it. This is the file that closed the
   measured shift.
2. **`src/app/layout.tsx`** — `Inter({ display: "optional" })` (was the implicit
   `"swap"` default). Belt-and-suspenders against font-swap reflow: `optional`
   gives Inter a ~100ms window and otherwise keeps the size-matched fallback
   (`adjustFontFallback`, on by default) with no later swap. Confirmed in built
   CSS: all 7 Inter faces now emit `font-display:optional`. Per Next 16.2.6
   `next/font` docs (`node_modules/next/dist/docs/01-app/.../font.md`).

Did NOT touch `Footer.tsx` (no change needed — footer already SSRs correctly),
`Header.tsx`, `AnimatedStatValue`, `globals.css`, or any page content.
