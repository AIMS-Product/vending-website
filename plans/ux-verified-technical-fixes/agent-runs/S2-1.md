# S2 — Public nav/footer tap targets ≥44px

Findings: C093 / C110 / C094 / C129 / C047 — public header nav links rendered
~20px tall and footer links ~17px tall, below the WCAG 2.2 AA 24px minimum and
the 44px comfortable mobile target.

## Outcome: PASS

Every header nav link and footer link now meets the targets. Header visual
height is unchanged (82px → 82px). Text size, weight, color, uppercase styling,
and the compact look are identical — only the invisible hit area grew.

## Files changed

- `src/components/site/Header.tsx` — desktop inline nav `HeaderNavLink` gains
  `py-3 -my-3` (the `-my-3` cancels the padding's layout flow so the 82px header
  height is unchanged). Mobile menu links already had `py-3` (48px) — untouched.
- `src/components/site/Footer.tsx` — each footer link becomes `block py-3` (44px
  box) and gains a `focus-visible` ring it previously lacked (matches the header
  link pattern; closes a keyboard-a11y gap on the newly enlarged target). The
  `<ul>` gap changes `space-y-3` → `-space-y-1.5` to pull the enlarged boxes back
  toward the original visual rhythm.
- `src/components/site/tap-targets.test.tsx` — new RGR unit test locking `py-3`
  onto nav anchors and `block py-3` onto footer anchors via static markup.
