## A11y notes

- Real `<button type="button">` elements → natively keyboard accessible.
- `aria-label` carries direction + block context (`Move block {N} up/down`, matching the neighboring `Add block below block {N}` pattern); `title` adds the block-type label (`Move Rich text up`).
- Tap targets `size-9` (36px), identical to the neighboring edit-settings and add-below buttons; same `focus-visible:ring-2 focus-visible:ring-sky-300` focus style; `disabled:opacity-35 disabled:cursor-not-allowed` with hover suppressed via `enabled:hover:*`.

## Residual risk

- Low. Boundary semantics are per-column (what `moveBlock` supports). On multi-column/multi-section pages, a block at a column edge cannot cross into the neighboring column/section from these buttons — same limitation as the existing canvas MovePositionMenu. If the tester expects cross-section movement from the outline, that's a new capability (moveBlockToIndex/cross-column action) out of this node's scope.
- Each outline row is now ~8px wider (one extra 36px control column was already budgeted by flex-1 min-w-0 on the label button; long summaries truncate as before). Visual check belongs to the orchestrator's browser gate.
