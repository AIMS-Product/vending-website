### Deviation note (ai-chat.ts line 113, context schema)

The node context listed `ai-chat.ts:106` (now :113) among the hard-180 sites, but that schema validates the _request context_ — the page's existing stored meta description sent to AI chat — not AI-written output. Capping it at 155 would have rejected the entire chat request for every legacy page holding a 156-180 char description, directly violating the "legacy content must not be blocked" constraint. It now uses `META_DESCRIPTION_LEGACY_MAX_LENGTH` (still de-literalized) with an inline comment. All genuine AI _generation_ schemas in that file (tool input schema + truncation limits) are at 155 as decided.
