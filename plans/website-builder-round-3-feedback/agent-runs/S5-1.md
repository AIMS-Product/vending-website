## Residual risk

- Publish-gate (`validatePageForPublish`) does not require alt text when a proof block has media (split-hero does). Out of scope per node instructions ("proofBlockSchema only; touch nothing else in this file"); flag for a follow-up decision if alt enforcement is wanted.
- `blockRegistry.proof.defaultProps` intentionally untouched — new proof blocks start without media props (valid, optional).
- Library proof-item picker stores `asset_id` on `proof_items`, but nothing yet auto-fills a proof _block_ from a library proof item's asset (block-level picker covers the editor path). Browser gate should confirm: editor canvas image, public render image, and `/admin/libraries` proof save with image.
- External media URLs in proof blocks rely on `next.config` image domains, same constraint as existing image/hero blocks.
