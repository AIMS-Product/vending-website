## Test idiom note

The node spec asked for Vitest + React Testing Library, but RTL is not installed in this repo and `vitest.config.ts` uses `environment: "node"`. The established component-test idiom is `renderToStaticMarkup` (e.g. `NewsHero.test.tsx`, `NewsArticle.test.tsx`). Per the instruction to match existing test setup/idiom, the test renders with `renderToStaticMarkup` and asserts on the extracted `class` attribute of the `<ul>`/`<ol>`. Adding RTL + jsdom would have been an out-of-scope dependency change.
