What Changed

  • Strengthened the AI page-builder system prompt around people-first SEO, non-thin structure, topic preservation, readable title casing, and no invented commercial claims.
  • Raised provider output caps from 5000 to 8000 tokens for both OpenAI Responses API and Cerebras chat completions.
  • Hardened media validation so a YouTube/video URL cannot be applied as an image block.
  • Added deterministic fallback body generation for metadata-only or risky full-page rebuilds, including existing-page replacement requests.
  • Exported assessReplacementSeoDraftQuality as a shared server-side quality gate with structured reasons and metrics.

Known Limitations

  • Live OpenAI parity could not be completed because the browser run showed OpenAI quota was unavailable. The current provider parity evidence is mocked request/response coverage plus live Cerebras browser success.
  • The browser sequence used the local admin editor surface and available provider state. It is strong manual evidence, but it is not the same as a committed CI e2e job.
  • SEO confidence is not a ranking guarantee. The changes improve content quality controls, topic specificity, and anti-spam guardrails.

How To Raise Confidence Further

  • Restore OpenAI quota, then rerun the same 10-topic browser matrix on both OpenAI and Cerebras.
  • Add a committed Playwright dependency and CI test that intercepts the AI chat API with fixtures, applies all block families in the editor, and asserts no rendering or autosave regressions.
  • Add a human editorial rubric for factual claims, local service coverage, legal/commercial phrasing, and publish readiness.
  • Track generated page quality over time with a stored scorecard for word depth, exact-keyword count, card/FAQ coverage, and risky phrase matches.

Final Position

The page builder is now substantially safer against thin SEO output. The biggest quality lift is the shared replacement draft assessment: it makes weak output measurable and automatically replaceable with a stronger deterministic draft.

Current release confidence is high for local prompt/gate behavior and block rendering. It should not be called provider-complete until live OpenAI quota is restored and parity is rerun.
